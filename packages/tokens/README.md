# @lumen/tokens

Design tokens generated from the Lumen-DS-2027 Figma file — the single
source of truth for color, typography, spacing, and radius. There is no
elevation/shadow tier; see `docs/figma-sync.md` for why.

## Editing tokens

Edit the JSON files in `src/`, never `dist/` directly:

- `src/primitives/color.json` — raw color scale (Figma primitives)
- `src/semantic/color.json` — light/dark semantic mapping (background, text, border, brand, status)
- `src/typography.json` — type scale (font family + Display/Headline/Title/Label/Body/Overline/Caption/Code sizes)
- `src/spacing.json` — layout scale (Figma-sourced) + finer component `space` scale, both keyed by literal pixel value
- `src/radius.json` — corner radius scale

Then rebuild:

```bash
pnpm --filter @lumen/tokens build
```

## What's generated (and committed)

`dist/` is committed to this repo (not gitignored) so downstream apps can
consume it as a git/path dependency without running a build step first. It
is regenerated by CI on every change and should never be hand-edited —
`scripts/build.mjs` will overwrite it.

- `dist/css/variables.css` — CSS custom properties, `:root` (light) +
  `[data-theme="dark"]`
- `dist/tailwind-preset.cjs` — Tailwind v3 preset mapping token names to the
  CSS variables
- `dist/index.ts` — typed JS/TS exports of every token, for use outside Tailwind

See `docs/figma-sync.md` at the repo root for which values are directly
sourced from Figma vs. engineering-extrapolated.
